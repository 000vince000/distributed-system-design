# System Design Practice Report

## Design Question

## Requirements
### Functional
- User authentication
- Data storage
- Real-time updates

### Non-functional
- High availability
- Low latency
- Data consistency

## APIs
### Internal
- POST /auth

### External
- GET /data

## Workflows
- POST /auth
  Requirement: User authentication
  Auth Service
    - Validate credentials
    - Generate token
  Database
    - Store session
    - Update last login
- GET /data
  Requirement: Data storage
  Load Balancer
    - Route request
    - Check health
  Application Server
    - Process request
    - Format response
  Cache
    - Check cache
    - Update cache
  Database
    - Query data
    - Return results

## Components
